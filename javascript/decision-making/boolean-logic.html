Comparisons

Moving onto comparisons now, using comparison operators, similar to how primitive operators work. Similar to how the plus sign or subtract operators work. These are built in to JavaScript but are not essentially for mathematical uses, I'm assuming they can be but I don't think they're fully for mathematic problems. Yeah so instead of doing math like adding two things together, these are used more or less to compare two things together.

List of Comparison Operators

  • > - Greater than.
  • < - Less than.
  • >= - Greater than or equal to.
  • <= - Less than or equal to.
  • == - Equality (Have the same value)
  • != - Not equal to.
  • === - Strict equality.
  • !== - Strict non-equality.

Double Equals (==)

Below is an explanation of double equals but honestly it's pretty confusing to me still.

  • Checks for equality of value, but not equality of type.
  • It coerces both values to the same type and then compares them.
  • This can lead to some unexpected results!
To put it simply, it checks if two values are equal, but doesn't care about the type (if they're the same). It will attempt to coerce two values, move them into being the same type which can give some unexpected results.

For example 1 == 1 will give me true because 1 and 1 are both numbers and they're the same value. However, when trying something like 1 == '1' then these two value types are not the same, one is a number and the other is a String. Though this will still give an output of true because they're pretty close, they're both still containing the value of one, they just don't contain the same value type. This can be problematic though because this operator doesn't care about type, because it converts them to the same type and THEN compares.
