JavaScript - Truthy & Falsy

This seems like an interesting part of JavaScript. Seems a little commical but at the same time interesting. First off these aren't really official terms for the way to describe this, but it's to do with how some values within JavaScript will be read as true and others as false.

A really good example of this would be the value "null" or "" (an empty String). These two would technically be counted as falsy by JavaScript. However, ALL JavaScript values inherit truthyness or falsyness. A list of falsyness values are show below.

  • false
  • 0
  • "" (empty String)
  • null
  • undefined
  • NaN

Reading that list must mean everything else is truthy, because they have an actual value? Well this is my own understanding for now so let's see what I've missed.

There's only one set of real true and false, but this concept does matter and will be noticeable when learning more. Every single value within JavaScript is considered truthy except from the ones listed above. Which in turn technically means if this value is needed in a condition then it's classed as false already because null is falsy. Which means that this code would be skipped over as if it was given an actual value and then came back as false.

I've added a script on here to see it in practice essentially. So firtly I'm creating a constant variable in which the it's looking for user input. If the user enters no input by hitting enter right away or hitting escape or cancel then the value of userInput would be "" (empty String). Which means that this input would be inheritly falsy from JavaScript.

The condition for this example is literally just (userInput) which JavaScript will read and use this Truthy and Falsy concept to figure out if what the user has entered is True or False. If the user has actually given some input then it would be Truthy and the there will be a console message saying this.

However, if they give nothing or somehow get one of the other values from the list above then it will be classed as Falsy and will print out Fasly in the console. Writing the condition with just the variable will check for this, no needing to use an operator for this built in inheritance check from JavaScript.

Another example would be very similar to the example above, but this time instead of checking userInput and for an empty String. The same will happen with the other values from the list, if they're passed into the conditional and put in the same place as the 0 was in the last example then they'd get the same output. Something to note is that any other number than 0 being sat in that condition would be Truthy, even negative numbers.

This seems to just be because JavaScript would want an actual value there to test before hand, but when given the list of values above then there's no point in checking them because they're already false as they're empty technically.
