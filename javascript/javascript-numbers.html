Operators

Javascript comes built in with some basic operations, similar to other languages, I can use things like addition, subtraction, multiplication, division, and modulo (which shows the remainder from a division basically). Just a little heads up on how Javascript does it's calculations with ordering, it goes like this, PEMDAS apparently. Which means parenthesis, exponents, multiplication, division, addition, and finally subtraction.

So there's something called the exponentiation operator within Javascript, and I don't remember seeing this being used anywhere prior. But basically this is how it looks firstly, "4 ** 6" which means 4 to the power of 6 which would give the result of 4096. (Totally didn't use the console to figure that out.)

NaN - Not a Number

So NaN is technically classed as a number within Javascript, it's something like a concept but it represents something that is not a number hence the name. Some examples are shown below:

  • 0/0 - This is a NaN.
  • 1 + NaN - This is also a NaN.
With the first example "0/0" this is impossible to do with normal numbers so the same applies to Javascript, there's something that this contributes to which is something I haven't touched on yet that is called "typeof", I have no idea how this works though. Apparently it's considered similar to an operator, like the plus sign for addition.

However, the difference with the "typeof" operator is that instead of needing 2 inputs like the others, this one only needs the one to work. When I type something like "typeof 4" into the console the console will print out "number" showing me that this is classed as a number, the same goes for "32.23221" this is also classed as a number. And guess what? typing in "NaN" also gives the output of number.

Knowing about NaN is useful for when trying to figure out what is going wrong with some code when an operation isn't giving the desired output and gives a NaN. It just means something went wrong somewhere. It's also useful to know that when using NaN in operations like the second example then it will also return NaN. Kind of like how X number divided by 0 always ends up at 0.
