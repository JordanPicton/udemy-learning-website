Touching on more about how to update variables, well mainly talking about numbers still for now. The way this works, is like this; "let score = 0" (the variable gets created with the value of 0), "score = 5" (then score gets given the value of 5), "score = score + 5" (then score is given the value of score which is currently 5 then 5 is added to this giving score the new value of 10). Everytime I want to add something to score I can do it like this and it can be another number like 21 for example which if I was to do that right now it would give me the total of 31. This also works for subtraction which would look like "score = score - 2" which in this case would give score the value of 8.

However, there is a easier and shorter way in doing this and it looks a little something like "score += 5" what this means is score, set it equal to score + 5. It's just the short hand of doing what I've been doing up above but condensed. And subtraction would look something like "score -= 5" which would take away 5 from score. There are other operators I can use in the same way, for example I've already shown addition +=, then subtraction -=, there's multiplication *=, and division /=.

i++ vs ++i

Depending on the increment or decrement operators placement it can change when the number value gets return, before before or after the increment/decrement. First off starting with i++, this is know as the post increment operator meaning you write the variable name and then the operator. When this is used it returns the current value of the variable and then increments it by one.

For example if I set the variable "i" to 0, "let i = 0". Then doing "i++" would print out 0 in the console, but it would have a value of 1 after printing this out. I've just tested it in the console to make sure I fully understood it and I do. Basically i++ prints the variable then increments. Meaing I don't get to see the updated number because the increment comes after the variable is printed. To fix this I can do ++i which baically increments i by 1 to the original value of i and then prints out the variable. Meaning I get to see the increment in real time rather than having to double check by typing the variable again. A little finicky but it's not too bad honestly since it is pretty straight forwards.

One more thing about this is that, when doing something like i++ (between the square brackets is the value given) this is what happens "let i = 5", "let result = i++", "result [5]", "i [6]", running "result" increments i by 1 but then I have to run i in console to check if it actually incremented i.

The better way to do this is the following "let i = 5", "let result = ++i", "result [6]", "i [6]". As shown I don't need to run i at the end to see the updated value since the value is printed out after the increment. Works for decrement too, kind of straight forward in that regard.

Const And Var

I've already set a variable with the keyword "let" like so "let variableName = 3;", but there are other ways I can variables, and I believe they each have their different uses.

Const

Starting with Const which stands for constant, it does exactly what you think it does, it's works in the same way let does so I can use it like "const variableName = 3;" the only difference being that I CANNOT change the value of this. So I can't increment or decrement this number or add other numbers to it etc. It's a constant meaning that it's constant, what a shocker.

Here are some examples of errors and things I cannot do with a constant. First off is trying to change the variable directly, for example; "const chairs = 4;", and then trying "chairs = 8;" this would give me an error bcause I can't change the constant. The same thing would happen if I was to try changing the value in the following way; "const people = 23;", "people = people + 1;", this would also error since I CANNOT change a constant! Another way I can't change this is by doing this "const dogs = 3;", "dogs += 1;", this again would not work.

The error I would get would be something like this "Type Error: Assignment to constant variable." The use case for const may seem very limited but it's actually really useful for when you want something to stay the same. Some good example of this are; "daysInWeek = 7;", "feetInMile = 5280;". There are use cases for these, just a good idea to keep them in mind.

When I go over Arrays and Objects, then I'll see more use cases where const would make more sense than using let. Javascript will yell at me for trying to change a const and will not let me.
