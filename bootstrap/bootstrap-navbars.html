First Navbar With Links

So it starts off like any other nav bar, you have the nav class as the main parent class and then an a link for the main element which is usually a logo or a brandname. Then the links come in and this is the part that can be different like I've showcased above.

After writing this all out, I'm a little confused honestly but I feel like I'll figure it out after the end of this lesson. Some of the classes or attributes used are a little confusing or haven't been explained yet, mainly talking about the aria attributes. But I believe there was one mentioned a while ago saying it had something to do with accessability. Yeah something that Colt says is that "Even though it's shorter, there's still a lot to do." I couldn't resonate with this more if I tried.

Some of the first few classes that are mentioned are "active" and "disabled". And I mean these first two are pretty straight forward. Active is to show which link is active in the navbar aka which link the user has currently selected. Where as disabled is the opposite basically, it's showing a link that cannot be selected. There are a few other classes mentioned which have to do with the a elements within the div. It seems like the a links need the classes "nav-link" and "nav-link". This is where I would also put the classes I mentioned before "active" and "disabled".

Something to note is that if the div with the class of "navbar-nav" was not a child of another div with the classes of "collapse" and "navbar-collapse" then the a links would not go across the navbar like how I'd expect they would go to the end of the navbar and act as if they were in a column. The collapse class is a plugin a part of Bootstrap that allows us to collapse and hide things, used on stuff like accordians etc. It can also be used in navbars to show and hide things. So to get the navbar to expand these links, I have to put a class inside the nav element called "navbar-expand-(size)". Giving a size to where I want to the navbar to expand. For example if I set it to navbar-expand-md then the navbar will expand the collapsed navbar to show the links at the medium breakpoint. And below it would stop showing these links. It's good for when there's a burger menu implemented for the smaller screens.

So for the burger menu that shows up is from the button which is nested within the main nav element. The button has the class of "navbar-toggler". I'm not sure this is what makes it tranform however, since I know some JavaScript goes into this to make it work, I'm sure I'll get shown this later in how this works. Button has a span with the class of "navbar-toggler-icon" inside of it which is basically the icon for the little toggleable menu. If I was just to include the button and the span with the classes I've given it then it wouldn't really do much since it hasn't been "linked" with the actual navbar that has the links. Meaning if I don't link it with the actual navbar it doesn't do anything but does display.

The way to connect the button with the navbar/links to extend when clicking on the button is by adding a "data-target" attribute with the same value as the id in the div that surrounds the navlinks. The only difference is that the data-target attribute has a # infront of the value. So for example id=testNav meaning that the data-target=#testNav. There is one other thing too with this however, that being a data-toggle attribute which is set collapse. Which is also a class within the same div around the navlinks. data-target is the id of something that we want it to expand of collapse. Also just as I thought, without Javascript this doesn't work.
